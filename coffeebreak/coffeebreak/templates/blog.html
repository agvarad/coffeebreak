My experiments with Google Datastore & Django models

Django ships with default Admin interface which was one reason that made me to tweek on Google Datastore. The Database engines are setup in settings.py file with the corresponding odbc modules.

Python-Django supported databases and handlers

1. SQLite ships with Django



2. MySQL with MySQLdb



3. Oracle with cx_Oracle



4. Postgresql with PyGreSQL



Once these modules are setup and config settings are done, uncomment the "django.contrib.admin" and "django.contrib.admindocs" from settings.py file. Now synch the db using "python manage.py syncdb" command. Django's admin site is ready for you.

Features of Django-Adminstration

1. User Registration.

2. Set priviliges for the users.

3. Create groups and group the users.

4. Manage your site contents with custom models.

About Google Datastore

Datastore is schemaless or a No SQL database which uses GQL as its querying language. Read more on Python and Datastore API @ Google Developers
